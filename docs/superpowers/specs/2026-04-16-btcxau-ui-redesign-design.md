# BTCXAU App UI 重做设计稿

## 1. 目标

本次 UI 重做的目标不是简单换皮，而是把当前 APP 重组为一套更统一、更像完整产品的深色交易应用。

本次设计以以下目标优先级为准：

- 品牌统一优先于信息堆叠
- 操作效率优先于模块数量
- 专业感优先于装饰感
- 读数效率在以上三点成立的前提下再优化

本次设计已经确认以下约束：

- 使用 `linear.app` 设计语言作为视觉参考
- 仅做深色主题，不维护同等优先级的浅色主题
- 不保留独立“仪表盘”页面
- 当前版本 APP 中“行情监控 -> 行情概览”模块不在新界面中体现
- 一级入口改为 `交易 / 账户 / 分析 / 设置`
- APP 默认进入 `交易` 页
- 顶部全局状态收敛为一个精简状态按钮
- 异常提醒整合进交易图表页，不再单独占一页或首页模块
- `分析` 页只做账户交易分析，不承接市场分析或异常复盘中心
- 二级内容采用混合打开方式：轻内容走底部弹层，重内容走新页面

## 2. 非目标

本次设计不包含以下目标：

- 不新增新的业务数据源
- 不新增新的账户分析算法
- 不恢复或兼容旧版“行情概览”展示逻辑
- 不把所有历史和分析内容重新堆回一级页
- 不引入与当前架构无关的新导航体系

## 3. 总体产品结构

新 APP 的一级结构固定为四个入口：

1. `交易`
2. `账户`
3. `分析`
4. `设置`

这四个入口的职责如下：

### 3.1 交易

作为 APP 默认首页和主工作台，负责承接：

- 当前品种与周期选择
- K 线图和指标
- 异常提示与异常图上标记
- 当前品种的仓位、挂单、风险摘要
- 买入、卖出、平仓、改止盈止损等交易操作

### 3.2 账户

作为账户管理页，负责承接：

- 账户总览
- 当前持仓
- 当前挂单
- 历史成交入口
- 单条持仓、挂单的详情入口

### 3.3 分析

作为账户交易分析页，负责承接：

- 收益曲线
- 回撤分析
- 胜率与盈亏比分布
- 持仓时长与交易结构分析
- 历史成交分析入口

### 3.4 设置

作为系统与偏好页，负责承接：

- 通知与提醒
- 悬浮窗与权限
- 显示与主题相关配置
- 连接状态、诊断、版本信息
- 其他系统设置

## 4. 导航与打开规则

### 4.1 一级导航

底部导航固定为四项：

- `交易`
- `账户`
- `分析`
- `设置`

不再保留独立“仪表盘”或“市场总览”页。

### 4.2 混合打开规则

为避免一级页过长，同时避免一切内容都跳深层页面，统一使用以下规则：

#### 轻内容

先走底部弹层：

- 全局状态
- 单条异常快看
- 单条持仓快看
- 单条挂单快看
- 单笔成交快看
- 单个指标口径说明

#### 重内容

直接进入新页面：

- 完整异常详情
- 完整持仓详情
- 完整挂单详情
- 完整历史成交列表
- 完整交易统计分析详情
- 各类结构分析详情页

### 4.3 一级页默认显示原则

一级页只展示“当前最需要看的摘要”，不展示完整长列表。

具体原则如下：

- 一级页默认展开主模块
- 一级页只展示摘要，不展示完整历史与完整分析
- 二级深内容必须下沉到新页面

## 5. 视觉系统

### 5.1 视觉方向

整体视觉以 `linear.app` 为参考，构建深色、冷静、精密、统一的产品感。

本次 UI 采用以下视觉原则：

- 深色原生，不做浅色主线
- 色彩尽量收敛，以冷灰和蓝紫强调为主
- 用层级、边框、透明度区分信息层，而不是大量彩色块
- 用统一卡片、按钮、标签、弹层体系建立品牌感

### 5.2 主题与色彩角色

建议以以下角色重建 Android 资源层：

- 页面底色：近黑蓝灰
- 面板底色：较页面底色略亮一阶
- 卡片底色：半透明深色卡片
- 主文字：高对比浅色
- 次文字：低一阶灰蓝
- 强调色：蓝紫色，仅用于关键交互与选中态
- 成功色：绿色，仅用于盈利、成功、正常状态
- 风险色：红色，仅用于异常、亏损、警告

### 5.3 组件风格

统一使用以下风格：

- 卡片：圆角、细描边、深色分层
- 底部导航：短标签、选中态高亮、避免旧式块状 tab
- 按钮：主按钮、次按钮、胶囊按钮三种层级
- 弹层：深色底部抽屉，和一级页同一视觉语言
- 图表工具：轻量胶囊按钮，不使用杂乱色块按钮条

## 6. 交易页设计

交易页是默认首页，也是全 APP 的主工作台。

### 6.1 交易页结构

从上到下固定为五层：

1. 顶部控制条
2. 风险提示条
3. K 线主图
4. 快捷交易区
5. 当前品种摘要

### 6.2 顶部控制条

显示：

- 当前品种
- 当前周期
- 指标按钮
- 图表工具按钮
- 全局状态按钮

交互规则：

- 品种、周期、指标在当前页内切换
- 不单独跳转新页面
- 工具类入口优先走轻菜单或弹层

### 6.3 风险提示条

显示：

- 当前最重要的一条异常或风险提示

交互规则：

- 默认只显示一条最重要提示
- 点击后先弹出异常快看层
- 再由用户决定是否进入完整异常详情

### 6.4 K 线主图

显示：

- K 线
- 指标
- 历史成交标注
- 当前持仓标注
- 当前挂单标注
- 异常点位标记

交互规则：

- 点异常标记：先弹异常快看层
- 点成交标记：先弹成交快看层
- 点持仓标记：先弹持仓快看层
- 深内容再进入对应详情页

### 6.5 快捷交易区

显示：

- 买入
- 卖出
- 平仓
- 改止盈止损

交互规则：

- 一级页直接可用
- 简单交易走弹层
- 复杂改单允许进入完整交易详情或完整交易弹窗

### 6.6 当前品种摘要

显示：

- 当前品种仓位摘要
- 当前挂单摘要
- 风险提示摘要
- 最近异常数量

交互规则：

- 默认只显示摘要
- 点某条对象先弹层
- 点“完整详情”进入对应新页面

## 7. 全局状态按钮设计

### 7.1 按钮默认显示

按钮本体只显示最简信息：

- 在线/离线
- 当前账户简称
- 当前异常数量

### 7.2 点击后的状态弹层

状态弹层显示：

- 连接状态
- 当前账户
- 同步状态
- 最近刷新时间
- 当前异常数量
- 快捷入口：去设置 / 去诊断 / 看异常列表

### 7.3 边界

状态按钮只承担：

- 全局状态查看
- 少量全局跳转

不承担：

- 复杂设置编辑
- 大段诊断内容浏览

这些内容必须下沉到设置页或诊断页。

## 8. 异常整合进图表的规则

异常不再独立占首页模块，而是并入交易图表页，用三层表达。

### 8.1 第一层：顶部风险提示条

用于表达当前最重要异常。

### 8.2 第二层：图上异常标记

用于把异常与具体时间和价格位置绑定。

### 8.3 第三层：异常列表入口

用于承接全部异常，不在交易页直接展开长列表。

### 8.4 统一交互顺序

统一交互顺序为：

1. 先在顶部看到风险提示
2. 再在图上看到异常位置
3. 点击异常标记或提示，先弹出异常快看层
4. 仍需更多内容时，再进入完整异常详情页

## 9. 账户页设计

账户页的目标是“先看懂当前账户状态”，不是一次铺开全部历史。

### 9.1 一级页默认可见模块

进入账户页时，默认展开并显示：

1. `账户总览`
2. `当前持仓`
3. `当前挂单`
4. `历史成交入口`

### 9.2 账户总览

显示：

- 净值
- 余额
- 保证金
- 可用保证金
- 浮盈亏
- 当日表现

交互规则：

- 默认展开
- 不需要额外点击展开
- 点某个指标只弹口径说明，不跳页

### 9.3 当前持仓

显示：

- 每个品种一张摘要卡
- 方向
- 数量
- 浮盈亏
- 开仓均价
- 风险提示

交互规则：

- 默认展开为摘要列表
- 点某条持仓先弹底部详情层
- 再点“完整详情”进入新页面

### 9.4 当前挂单

显示：

- 品种
- 类型
- 价格
- 数量
- 下单时间

交互规则：

- 默认展开为摘要列表
- 点某条挂单先弹底部详情层
- 复杂编辑和完整查看进新页面

### 9.5 历史成交入口

显示：

- 最近几条成交
- 时间范围摘要
- 总成交数摘要

交互规则：

- 一级页不展开完整长列表
- 点击后进入完整历史成交整页

## 10. 分析页设计

分析页只做账户交易分析，不承接市场分析中心角色。

### 10.1 一级页默认可见模块

进入分析页时，默认显示：

1. `收益曲线`
2. `核心统计摘要`
3. `结构分析卡片`
4. `历史成交分析入口`

### 10.2 收益曲线

显示：

- 净值曲线
- 结余曲线
- 当前选定时间范围
- 关键回撤区间提示

交互规则：

- 默认展开
- 作为分析页主视觉中心
- 时间范围和曲线类型在页内切换

### 10.3 核心统计摘要

显示：

- 总收益
- 最大回撤
- 胜率
- 盈亏比
- 平均持仓时长
- 交易次数

交互规则：

- 默认只显示摘要
- 点指标先弹轻说明层
- 不单独跳转新页面

### 10.4 结构分析卡片

显示：

- 交易分布
- 持仓时长分布
- 月度/周期表现摘要
- 多空表现对比摘要

交互规则：

- 一级页先只展示摘要卡
- 点击进入对应完整分析详情页

### 10.5 历史成交分析入口

显示：

- 最近复盘对象
- 异常成交提示
- 逐笔分析入口

交互规则：

- 点某笔成交先弹底部快看层
- 再点“完整分析”进入该笔成交详情页

## 11. 设置页设计

设置页承接所有系统级与偏好级内容。

建议按以下分组：

1. 通知与提醒
2. 悬浮窗与权限
3. 显示与界面
4. 连接、诊断与版本
5. 其他系统设置

设置页不承担交易、账户、分析业务内容。

## 13. 组件与实现边界

本次实现应优先重建下列可复用 UI 组件，而不是逐页零散改颜色：

1. 全局深色主题 token
2. 一级卡片组件
3. 深色底部弹层样式
4. 顶部控制条组件
5. 状态按钮组件
6. 风险提示条组件
7. 列表摘要卡组件
8. 分析摘要卡组件

## 14. 实施顺序建议

建议按以下顺序推进实现：

1. 先重建主题 token、颜色、文字层级、卡片和弹层基础组件
2. 先改主壳与底部导航
3. 再完整重做 `交易` 页
4. 再重做 `账户` 页
5. 再重做 `分析` 页
6. 最后收口 `设置` 页和全局状态按钮

原因：

- `交易` 页是新的默认首页，必须最先成立
- `账户` 和 `分析` 的结构已经清晰，后续迁移风险更低
- 先做基础组件再做页面，能保证整套界面统一

## 15. 验收标准

本次 UI 设计的验收标准如下：

### 15.1 产品结构

- 不再出现独立仪表盘页
- 不再出现旧版“行情概览”模块
- 一级入口固定为 `交易 / 账户 / 分析 / 设置`

### 15.2 页面边界

- 交易页承担图表、异常、交易操作
- 账户页承担当前账户与对象管理
- 分析页承担账户交易分析
- 设置页承担系统设置与诊断

### 15.3 交互边界

- 轻内容优先弹层
- 重内容进入新页面
- 一级页不铺开完整长列表

### 15.4 视觉一致性

- 全 APP 统一深色视觉语言
- 卡片、按钮、弹层、导航风格一致
- 不再出现旧页风格和新页风格混杂

## 16. 已确认决策

以下决策已经在本轮设计讨论中确认，不再重复讨论：

- 全量重做 UI，而不是只换视觉层
- 深色唯一主线
- 默认首页为 `交易`
- 不保留独立 `仪表盘`
- 不保留旧版“行情概览”
- 顶部全局状态收为一个状态按钮
- 异常提醒整合进图表页
- 分析页只做账户交易分析
- `账户页` 与 `分析页` 的一级结构采用“默认可见主模块 + 弹层快看 + 深页详情”
