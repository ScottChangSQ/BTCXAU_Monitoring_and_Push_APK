# Notion Data Desk UI 正式设计稿

## 1. 目标

本轮目标是把当前 App 的网页视觉方向正式收口为 `Notion Data Desk`。

约束已经确认：

- 不删除现有功能
- 尽量不重排现有页面结构
- 只做轻量整理，不重新发明信息架构
- 优先保留当前交易类页面的扫读效率

本轮不是做“像 Notion 官网那样的大留白品牌页”，而是做：

- 暖白底
- 高密度
- 文档式
- 数据工作台

也就是把当前交易产品的结构保留住，再把视觉语言换成更克制、更统一、更容易长期扩展的一套浅色系统。

## 2. 结构边界

本轮默认保持现有页面和功能入口不变：

- 主壳：`交易 / 账户 / 分析`
- 设置：独立设置目录页
- 交易页：顶部控制区 + 图表区 + 指标带
- 账户页：账户总览 + 当前持仓 + 挂单 + 历史入口
- 分析页：摘要统计 + 图表 + 分布图
- 弹层：全局状态、异常列表、历史相关弹层、交易弹窗
- 悬浮窗：小尺寸运行态卡片

不做的事：

- 不新增一级页面
- 不删除当前已有功能
- 不改主页面顺序
- 不把高频交易入口藏到更深层
- 不把页面改成“品牌展示页”式的大面积留白布局

## 3. 设计方向结论

本轮最终确认方向为 `C. Notion Data Desk`。

这条方向的核心不是“更轻”，而是“更稳”：

- 比 BMW 深色版更温和
- 比纯 Notion 官网风格更高密度
- 比旧金融工具风格更统一

设计关键词：

- 温暖中性
- 高密度但不拥挤
- 轻边框
- 白纸式模块
- 单一蓝色高亮
- 盈亏继续保留红绿语义

## 4. 视觉系统

### 4.1 基础底色

| 角色 | 颜色 | 用途 |
|---|---|---|
| 全局背景 | `#f6f5f4` | 页面底层暖白背景 |
| 主面板 | `#ffffff` | 卡片、模块、弹层主体 |
| 次面板 | `#fbfaf9` | 轻分层、小控件背景 |
| 主文字 | `rgba(0,0,0,0.95)` | 标题、主数据 |
| 次文字 | `#615d59` | 标签、说明、普通文本 |
| 弱文字 | `#a39e98` | 时间、辅助文案、弱状态 |
| 主边框 | `rgba(0,0,0,0.10)` | 卡片、按钮、分割线 |
| 强边框 | `rgba(0,0,0,0.18)` | 需要提高存在感的边框 |

### 4.2 交互色

| 角色 | 颜色 | 用途 |
|---|---|---|
| 主高亮蓝 | `#0075de` | 当前选中、关键操作、聚焦 |
| 蓝色浅底 | `rgba(0,117,222,0.10)` | 选中态背景 |
| 正收益 | `#1f9d55` | 盈利、上涨、正向提示 |
| 负收益 | `#d94841` | 亏损、下跌、风险提示 |
| 警示 | `#dd5b00` | 中性提醒、阈值、预警说明 |

约束：

- 蓝色只用于交互高亮，不做大面积品牌铺底
- 红绿只用于交易语义，不参与普通按钮装饰

### 4.3 圆角与阴影

- 主模块圆角：`16px`
- 外壳圆角：`18px - 22px`
- 小按钮圆角：`10px`
- 胶囊标签：全圆
- 阴影保持很轻，只表达层次，不表达“悬浮感”

### 4.4 字体与层级

排版目标：

- 标题更紧凑
- 文案更克制
- 数据更突出
- 辅助信息更后退

建议层级：

| 角色 | 字号建议 | 字重建议 |
|---|---|---|
| 页面大标题 | `34px - 60px` | `700` |
| 模块标题 | `15px - 24px` | `700` |
| 一级操作按钮 | `12px` | `700` |
| 二级切换按钮 | `11px` | `600` |
| 正文说明 | `12px - 15px` | `400 - 600` |
| 主数据 | `24px` 左右 | `700` |

## 5. 组件规则

### 5.1 顶部一级控件

交易页顶部 4 个控件保持原位置：

- `BTCUSDT`
- `交易`
- `挂单`
- `已连接 | 账户在线`

本轮已确认的视觉分组规则：

- `BTCUSDT` 与 `已连接 | 账户在线` 使用同一套高亮样式
- `交易` 与 `挂单` 使用同一套普通样式

原因：

- 左右两端都属于“当前状态/当前上下文”
- 中间两项属于“模式切换”

这能让顶部 4 项的关系更清楚，不再像 4 个彼此独立的按钮。

### 5.2 切换条与轻操作按钮

- 周期切换、指标切换、分析页分段筛选统一用轻边框样式
- 当前项用蓝色浅底 + 蓝色强调边界表达
- 非当前项保持白底或暖白底
- 不使用实心重色块压过图表或正文

### 5.3 模块与列表

- 模块统一用白底卡片
- 列表项用更轻的白底或暖白底
- 通过细边框和留白建立层次
- 不靠大量投影、渐变和发光区分模块

### 5.4 弹层与对话框

底部弹层和弹窗统一规则：

- 白底或暖白白底
- 轻边框
- 圆角明显但不过分夸张
- 操作按钮和页面主按钮共用一套视觉系统

## 6. 页面映射

### 6.1 交易页

保留现有结构：

- 顶部 4 控件
- 图表主场
- 右上角轻量摘要
- 左下角图层开关
- 底部指标切换带

本轮已确认的细节：

- 删掉交易页顶部额外说明模块，不再显示“交易主场”说明卡
- 图表继续保留高密度，不改成大留白内容页
- 右上角持仓摘要保留，仍作为图表内轻量信息区
- 历史成交与当前仓位开关继续保留在图表里

视觉目标：

- 更像数据工作台
- 更像研究面板
- 不像旧版深色终端
- 也不像过度品牌化网页

### 6.2 账户页

保留现有结构：

- 账户总览
- 当前持仓
- 挂单
- 交易记录入口

视觉目标：

- 总览卡更像“账户摘要卡”
- 持仓和挂单更像“数据行”
- 操作按钮明确但不喧宾夺主

### 6.3 分析页

保留现有结构：

- 核心统计
- 时间维度切换
- 交易统计
- 散点分布

视觉目标：

- 仍然像分析面板
- 不变成内容文章页
- 图表容器统一成同一套轻卡片系统

### 6.4 设置页

设置页当前已确认的层级调整：

- `悬浮窗与显示`
- `MT5 网关设置`
- `主题设置`
- `缓存管理`
- `运行日志`

这 5 项保持一级并列关系。

不再使用：

- `辅助工具 Tools` 作为额外分组容器去包住 `缓存管理`、`运行日志`

原因：

- 这两个入口对当前产品来说不是“更低一级工具”
- 它们和上面的设置项在阅读层级上应保持一致

### 6.5 全局状态弹层

保留现有结构：

- 连接状态
- 当前账户
- 同步状态
- 最近刷新
- 异常数量
- `设置 / 日志 / 异常列表`

视觉目标：

- 看起来像系统状态面板
- 不是黑色运维控制台
- 数据和值要有明显层级，但整体仍然简洁

### 6.6 异常列表弹窗

本轮已确认要纳入正式设计稿和网页稿。

弹窗包含：

- 标题：`异常列表`
- 数量统计
- 当前阈值展示
- 实时监控状态
- 异常记录列表
- 动作按钮：`阈值设置 / 关闭`

每条异常记录建议结构：

- 产品名
- 方向或变化值
- 时间
- 简短异常说明

视觉规则：

- 单条记录使用独立轻卡片
- 正负变化继续保留红绿语义
- 不使用深色告警面板风格
- 让异常信息更容易扫读，而不是更紧张

### 6.7 悬浮窗

保留现有结构：

- 顶部盈亏
- 连接状态
- 产品卡片

视觉目标：

- 小而稳
- 信息集中
- 不做复杂装饰
- 与主页面保持同一套暖白数据工作台语言

## 7. 网页稿输出要求

正式网页示意图位于：

- `docs/superpowers/mockups/2026-04-17-notion-data-desk-mockup.html`

网页稿当前应覆盖：

- 交易页
- 账户页
- 分析页
- 设置页
- 全局状态弹层
- 悬浮窗
- 异常列表弹窗

用途：

- 用于确认视觉方向
- 用于给后续真实实现提供对照
- 不直接承担业务实现

## 8. 实现边界

本轮设计稿只固化视觉方向，不直接改业务逻辑。

后续真实实现时应遵守：

- 不因为换视觉而删除已有功能
- 不为了“更像 Notion”去牺牲高频交易扫读效率
- 不把浅色化做成低对比、难读的展示页
- 不额外引入新的一级流程

## 9. 本轮结论

本轮最终确认的不是“Notion 官网风格”，而是：

- 保留当前交易产品结构
- 保留当前数据密度
- 用 Notion 式暖白、轻边框、单蓝色强调重做视觉系统

它的正确目标是：

- 比旧深色版更容易长期维护
- 比品牌展示页更适合交易产品
- 比原始工具风格更统一、更高级
