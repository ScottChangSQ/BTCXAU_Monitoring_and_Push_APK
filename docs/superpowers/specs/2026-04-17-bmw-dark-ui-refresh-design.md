# BMW 深色 showroom 版 UI 设计稿

## 1. 目标

本轮只更新视觉，不调整现有页面结构、信息层级和功能入口。  
目标是把当前 App 从“终端感深色界面”收口为更接近 BMW 官网深色段落的气质：

- 更硬朗
- 更克制
- 更工业精密
- 更像高端控制台，而不是工具台

保留现有结构：

- 主壳：`交易 / 账户 / 分析`
- 交易页：顶部控制区 + 图表主场 + 指标带
- 账户页：总览 / 持仓 / 挂单 / 历史
- 分析页：多张统计卡纵向长页
- 设置页：列表式入口
- 弹层：全局状态、历史成交、异常列表、交易弹窗
- 悬浮窗：小尺寸运行态卡片

## 2. 总体视觉方向

### 2.1 气质

采用 BMW 深色 showroom 版，而不是 BMW 官网“深浅交替”的整站结构。

落地规则：

- 全局改为近黑底与深灰面板
- 模块之间不用阴影堆层次，主要靠底色层级和细边框分层
- 所有边角保持 0dp 直角
- 蓝色只用作交互高亮、焦点、选中状态、链接和关键操作提示
- 标题更克制，字重拉开层级，不靠装饰色吸引注意

### 2.2 结构约束

不改下列结构：

- 不新增一级页面
- 不挪动原有操作入口位置
- 不改变交易页、账户页、分析页的主要块顺序
- 不改变弹层打开方式
- 不改变悬浮窗尺寸逻辑

## 3. 色彩系统

### 3.1 基础色

| 角色 | 颜色 | 用途 |
|---|---|---|
| App 背景 | `#0A0C0F` | 全局最底层背景 |
| 主面板 | `#13171C` | 卡片、模块、主内容区 |
| 次面板 | `#1A2027` | 按钮、输入框、筛选条、次级容器 |
| 高亮面板 | `#202833` | 激活但未强选中的区域 |
| 主边框 | `#2C333C` | 模块边框、分隔线 |
| 强边框 | `#495261` | 焦点外框、重要分隔 |

### 3.2 文本色

| 角色 | 颜色 | 用途 |
|---|---|---|
| 主文字 | `#F5F7FA` | 标题、主数据、主要按钮文字 |
| 次文字 | `#B3BAC4` | 说明、标签、未选中控件文字 |
| 弱文字 | `#7E8793` | 时间、补充状态、说明信息 |
| 失效文字 | `#5B636E` | 禁用态 |

### 3.3 交互色

| 角色 | 颜色 | 用途 |
|---|---|---|
| BMW Blue | `#1C69D4` | 选中、聚焦、链接、激活态 |
| Focus Blue | `#0653B6` | 焦点描边、按下态 |
| 正收益 | `#3FCB7F` | 盈利、上涨、正向仓位结果 |
| 负收益 | `#D95C5C` | 亏损、下跌、风险提示 |
| 警示 | `#D7A64A` | 中性提醒、同步中、等待态 |

说明：

- BMW 蓝不铺大面积背景，只做交互高亮
- 绿红只保留给交易语义，不参与普通 UI 装饰

## 4. 字体与层级

由于当前 App 为 Android 原生界面，本轮不强制替换为 BMWType。  
网页示意图与后续 App 落地统一采用“接近 BMW 轻标题 + 重导航”的层级关系。

### 4.1 层级规则

| 角色 | 建议字重 | 说明 |
|---|---|---|
| 页面主标题 | 300 / 400 | 更轻、更克制，必要时大写 |
| 模块标题 | 400 | 清晰但不笨重 |
| 主导航 / 关键标签 | 700 / 800 | 用于底部 tab、主切换控件 |
| 正文 / 标签 | 400 | 常规文本 |
| 按钮文字 | 700 | 操作感更明确 |

### 4.2 排版原则

- 行高整体偏紧
- 交易与账户主数字允许更大字号
- 小控件文字不再走“终端式亮色小字”，改为金属灰 + 白字体系
- 只在强操作入口使用更重字重

## 5. 组件规则

### 5.1 按钮

统一规则：

- 全部直角
- 无圆角
- 无发光
- 无厚阴影
- 按键背景不用颜色渐变
- 同一级按钮的高度、字号、字重、文字颜色保持一致
- 同一级选项控件的激活方式保持一致，只通过纯色背景、细边框、底边蓝线来区分状态

按钮分 3 类：

1. 主动作按钮
- 背景：`#1A2027`
- 文字：`#F5F7FA`
- 边框：`1px #495261`
- 激活：底边或外边框改为 `#1C69D4`

2. 次动作按钮
- 背景：透明或 `#13171C`
- 文字：`#B3BAC4`
- 边框：`1px #2C333C`

3. 当前选中按钮
- 背景仍保持深色，不整块染蓝
- 文字变白
- 底边 / 左边 / 外框使用 BMW Blue

### 5.1.1 按钮层级规范

为避免同级控件尺寸和字体不统一，后续落地统一按 3 个层级执行：

| 层级 | 用途 | 高度 | 字号 | 字重 | 文字色 |
|---|---|---|---|---|---|
| L1 主切换按钮 | 顶部 `产品 / 交易 / 挂单 / 状态`、底部主导航 | `42dp` | `12sp` | `700` | 未选中 `#B3BAC4`，选中 `#F5F7FA` |
| L2 次级选项按钮 | 周期、指标、分析页分段筛选、弹层轻量选项 | `30dp` | `11sp` | `600` | 未选中 `#B3BAC4`，选中 `#F5F7FA` |
| L3 动作按钮 | `平仓 / 修改 / 设置 / 日志 / 异常列表 / 查看完整历史成交` | `34dp` | `11sp` | `700` | 主动作 `#F5F7FA`，次动作 `#F5F7FA` |

补充规则：

- 同一行里如果属于同一层级，不能出现一个 34dp、一个 36dp、一个 40dp 的情况
- 同一层级不再混用不同字重
- 同一层级不再混用不同主文字颜色
- 宽度可以按内容和布局变化，但高度和文字规则不变

### 5.2 下拉菜单

- 折叠态用深色矩形面板
- 打开后菜单背景比主面板再浅一层
- 当前项左侧增加蓝色细条或蓝色描边
- 文字保持白 / 灰，不使用高饱和背景块

### 5.3 卡片 / 模块

- 模块背景：`#13171C`
- 内嵌小块：`#1A2027`
- 主要靠细边框和上下留白建立层次
- 不使用温暖色背景，不使用玻璃拟态

### 5.4 抽屉 / 底部弹层 / 弹窗

- 背景：`#11151A`
- 边框：顶部或外轮廓 `1px #2C333C`
- 标题更轻，正文更稳
- 动作区按钮遵循主动作 / 次动作规则
- 弹层里的同级按钮高度必须一致，不因文案长度改变高度

### 5.5 边框与分割线

- 所有边框尽量细
- 统一金属灰
- 比当前“终端块”更像精密装配件

### 5.6 图表区

- 图表主底：更深的黑灰底
- 网格线更细、更克制
- 价格轴、时间轴文字改为弱文字灰
- 激活元素与操作控件的高亮统一用 BMW Blue
- 盈亏与买卖标注继续保留语义色

## 6. 各界面映射

### 6.1 交易页

现有结构保持不变：

- 顶部：产品、交易、挂单、状态
- 中部：周期带、图表主场、叠加摘要
- 底部：指标带

视觉更新：

- 顶部 4 控件像一组汽车中控切换键
- 状态按钮不再像普通按钮，更像状态条目
- 图表卡片边界更清楚，右上摘要更像 HUD
- 周期与指标切换条改成“细边框 + 蓝色选中线”的精密控件风格
- 顶部 4 控件统一按 `L1` 层级执行
- 周期与指标按钮统一按 `L2` 层级执行

### 6.2 账户页

现有结构保持不变：

- 账户总览
- 当前持仓
- 挂单
- 交易记录入口

视觉更新：

- 总览卡更像仪表总成
- 持仓条目和挂单条目更像清单行，不再像“功能块”
- 右侧操作按钮变成工业式矩形动作键
- 盈亏数据突出，但不使用大片彩色底
- `平仓 / 修改 / 筛选 / 查看完整历史成交` 统一按 `L3` 层级执行

### 6.3 分析页

现有结构保持不变：

- 收益类卡片
- 统计切换控件
- 柱状图、散点图、分布图

视觉更新：

- 更像车辆性能分析台
- 图表标题、说明、切换更收敛
- 图表容器深色统一，减少目前“模块之间像不同套件”的割裂感
- 分析页分段切换统一按 `L2` 层级执行

### 6.4 设置页

现有结构保持不变：

- 分组标题
- 列表入口

视觉更新：

- 更像车机设置菜单
- 分组标题为弱文字
- 行项为深色列表面板 + 细分隔
- 箭头与辅文统一金属灰

### 6.5 全局状态弹层

现有结构保持不变：

- 状态键值
- 设置 / 日志 / 异常列表

视觉更新：

- 更像系统面板，而不是普通弹窗
- 数值区更有秩序
- 三个按钮像并排功能键，选中或聚焦时才用蓝色强调
- 三个操作按钮统一按 `L3` 层级执行

### 6.6 悬浮窗

现有结构保持不变：

- 顶部盈亏
- 连接状态
- 产品卡片

视觉更新：

- 更像小型仪表 HUD
- 小面积深色卡片 + 白字 + 金属灰说明
- 盈亏保持绿红，但其余交互高亮使用 BMW Blue

## 7. 网页 mockup 输出要求

本轮网页示意图用于表达视觉方案，不用于实现业务。

输出要求：

- 单页 HTML
- 以网页方式展示多个手机界面示意
- 至少包含：交易页、账户页、分析页、设置页、全局状态弹层、悬浮窗
- 使用 BMW 深色 showroom 版视觉规则
- 明确保留当前 App 结构，不重新发明布局

## 8. 不做的事

- 不修改 App Java / XML 业务代码
- 不改实际布局结构
- 不新增页面流程
- 不引入暖色主背景
- 不把 BMW 蓝用成大面积铺底色
- 不在按键背景中使用颜色渐变

## 9. 本轮结论

本轮设计的核心不是“把 App 做成 BMW 官网”，而是：

- 继承 BMW 的深色工业精密感
- 保留当前交易产品的高密度信息结构
- 用更克制、更高级、更统一的方式重做配色与控件观感
