# 交易页快捷交易与挂单模式设计

## 1. 背景

当前交易页已经把主场收口到 K 线图，但交易入口仍然分散：

- 顶部只有产品选择和状态按钮，没有直接切换交易类型的入口
- 页面下方仍保留旧的“快捷交易模块”
- 旧桥接交易页还保留“成交量 / 成交额”摘要模块
- 挂单仍依赖旧弹窗表单，不能在图表主场直接完成价格选择和下单

这会让交易链路重复、视觉层级分散，也不符合“交易页以图表为主场”的目标。

## 2. 目标

本轮设计的目标只有一个：把交易页收口成“图表主场 + 顶部模式切换 + 图表内快捷执行”。

具体表现为：

- 顶部新增 `交易`、`挂单` 两个模式按钮，放在“产品选择”和“状态按钮”之间
- 同一时间只允许展开一种模式
- `交易` 模式提供一行快捷市价操作条
- `挂单` 模式提供一行快捷挂单操作条，并在图表内显示可拖动价格线
- 点击快捷按钮后直接走真实执行链，不再弹出旧下单表单
- 删除旧“快捷交易模块”
- 删除旧桥接交易页里的“成交量 / 成交额”摘要模块

## 3. 范围

### 本次包含

- 交易页顶部按钮结构调整
- 交易 / 挂单互斥模式状态
- 图表上方快捷操作条
- 市价买入 / 卖出快捷执行
- 图表内挂单价格线
- 挂单买入 / 挂单卖出快捷执行
- 根据价格线和当前价自动判断 `Buy Limit / Buy Stop / Sell Limit / Sell Stop`
- 删除旧快捷交易模块
- 删除旧桥接交易页成交量 / 成交额摘要模块
- 补齐相关测试

### 本次不包含

- 快捷模式里的止损 / 止盈输入
- 复杂改单、撤单、部分平仓的新快捷入口
- 重新设计旧交易确认弹窗
- 新增新的后端协议或新的交易真值链

## 4. 用户界面

### 4.1 顶部结构

交易页顶部第一行从左到右调整为：

`产品选择` | `交易` | `挂单` | `状态按钮`

要求：

- `交易` 和 `挂单` 都使用和当前顶部轻按钮一致的样式体系
- 两者是互斥模式按钮
- 未展开时两者都处于普通态
- 展开中的按钮使用选中态

### 4.2 快捷操作条

快捷操作条放在：

- 顶部按钮行下方
- 周期选项行上方

这样保持“先选交易模式，再看图，再执行”的阅读顺序，不插到图表下方重新形成第二个主操作区。

#### 交易模式

显示一行：

`买入` | `手数输入框` | `卖出`

要求：

- 默认手数为 `0.05`
- 点击 `买入` 后按输入手数直接执行市价买入
- 点击 `卖出` 后按输入手数直接执行市价卖出

#### 挂单模式

显示一行：

`挂单买入` | `手数输入框` | `挂单卖出`

要求：

- 默认手数为 `0.05`
- 图表内同时显示横向挂单价格线
- 点击 `挂单买入` / `挂单卖出` 后，以当前价格线价格作为挂单价直接执行

### 4.3 模式切换规则

- 当前关闭时，点 `交易` -> 展开交易模式
- 当前关闭时，点 `挂单` -> 展开挂单模式
- 当前已在 `交易` 模式时，再点 `交易` -> 收起
- 当前已在 `挂单` 模式时，再点 `挂单` -> 收起，并隐藏价格线
- 当前已在 `交易` 模式时，点 `挂单` -> 收起交易，展开挂单，并显示价格线
- 当前已在 `挂单` 模式时，点 `交易` -> 隐藏价格线，收起挂单，展开交易

## 5. 执行规则

### 5.1 当前有效价

快捷交易和快捷挂单都需要一份统一的“当前有效价”。

解析顺序：

1. 当前图表实时尾价
2. 当前窗口最新一根 K 线收盘价
3. 若都不存在，则不允许快捷执行

当拿不到当前有效价时，页面提示：

`当前价格未就绪，暂时不能执行交易`

### 5.2 市价买入 / 卖出

点击 `买入` 或 `卖出` 时：

1. 读取当前品种
2. 读取手数输入
3. 读取当前有效价
4. 生成现有 `TradeCommand`
5. 走现有 `check -> submit -> result -> refresh` 执行链

命令规则：

- `买入` -> `OPEN_MARKET + buy`
- `卖出` -> `OPEN_MARKET + sell`
- 快捷模式默认不带 `SL / TP`

### 5.3 挂单价格线

进入挂单模式时：

- 图表中显示一条横向价格线
- 初始价格取当前有效价
- 用户可以上下拖动修改价格

退出挂单模式时：

- 价格线立即隐藏

价格线要求：

- 只在价格区域工作
- 不覆盖或破坏现有持仓 / 挂单 / 历史成交标注
- 拖动后实时回传当前选中的价格

### 5.4 自动挂单类型判断

挂单模式执行时，根据“价格线相对当前有效价的位置”自动判断挂单类型：

#### 点击挂单买入

- 价格线低于当前价 -> `buy_limit`
- 价格线高于当前价 -> `buy_stop`

#### 点击挂单卖出

- 价格线高于当前价 -> `sell_limit`
- 价格线低于当前价 -> `sell_stop`

若价格线与当前价接近到无法稳定判断，则拒绝执行并提示：

`请把挂单线再上移或下移一点`

### 5.5 快捷挂单执行

点击 `挂单买入` / `挂单卖出` 时：

1. 读取当前品种
2. 读取手数输入
3. 读取价格线价格
4. 读取当前有效价
5. 自动判断挂单类型
6. 生成现有 `TradeCommand`
7. 走现有 `check -> submit -> result -> refresh` 执行链

命令规则：

- 复用现有 `PENDING_ADD`
- 复用现有 `TradeCommandFactory.pendingAdd(...)`
- `orderType` 只在四类中选：
  - `buy_limit`
  - `buy_stop`
  - `sell_limit`
  - `sell_stop`

## 6. 页面状态

新增一个明确的交易模式状态，而不是在页面里散落多个布尔值。

推荐状态：

- `CLOSED`
- `MARKET`
- `PENDING`

页面还要维护以下轻量状态：

- 当前手数文本
- 当前挂单线价格

状态原则：

- 手数在 `交易 / 挂单` 模式之间共用，不切换时清空
- 价格线只在 `PENDING` 模式下存在
- `CLOSED` 模式下不显示快捷操作条，也不显示价格线

## 7. 模块拆分

### 7.1 页面壳层

文件：

- `MarketChartScreen`

职责：

- 顶部按钮装配
- 模式切换
- 快捷操作条显示 / 隐藏
- 手数输入框与价格线状态同步

不负责：

- 自动判断挂单类型
- 生成交易命令
- 直接处理交易执行分支

### 7.2 图表交互层

文件：

- `KlineChartView`

职责：

- 新增挂单参考线绘制
- 处理参考线上下拖动
- 把拖动后的价格回传给页面

不负责：

- 下单判断
- 执行链

### 7.3 快捷交易协调层

新增文件建议：

- `ChartQuickTradeCoordinator`
- `ChartQuickTradeMode`

职责：

- 校验快捷模式输入
- 解析当前有效价
- 自动判断挂单类型
- 生成现有 `TradeCommand`
- 调用现有 `TradeExecutionCoordinator`

### 7.4 执行与刷新层

继续复用：

- `TradeExecutionCoordinator`
- `TradeCommandFactory`
- 现有账户刷新链
- 现有图表叠加刷新链

这样可以保证：

- 不复制交易真值链
- 不并行维护第二套交易提交逻辑
- 快捷入口和旧弹窗入口最终落到同一条底层主链

## 8. 删除项

### 8.1 交易页旧快捷交易模块

必须删除，不保留“新快捷条 + 旧快捷卡”双入口。

### 8.2 旧桥接交易页成交量 / 成交额模块

必须删除，避免旧桥接页面仍然保留已不符合当前交易页信息架构的摘要模块。

## 9. 错误处理

### 输入问题

- 手数为空 -> 提示必须填写手数
- 手数小于等于 0 -> 提示必须大于 0
- 当前价不存在 -> 提示当前价格未就绪
- 挂单线价格不存在 -> 提示挂单价格未就绪
- 挂单线与当前价无法区分 -> 提示请调整挂单线位置

### 执行问题

- `check` 失败 -> 展示服务端返回原因
- `submit` 失败 -> 展示失败原因，不收起模式
- 结果未确认 -> 保持当前模式并提示未确认，不伪装成成功

### 成功后

- `交易` 模式保持展开
- `挂单` 模式保持展开，价格线保留
- 统一刷新持仓、挂单、图表叠加和右上角持仓摘要

- 顶部按钮结构

- 删除旧快捷交易模块
- 删除旧桥接页成交量 / 成交额模块

### 步骤 2

再接模式切换：

- `CLOSED / MARKET / PENDING`
- 交易 / 挂单互斥展开

### 步骤 3

接快捷市价交易：

- 买入

### 步骤 4

## 12. 关键决定

### 决定 1：不复用旧交易弹窗作为快捷入口

原因：

- 用户要的是图表主场内的一键操作
- 弹窗会把“快捷模式”重新拉回多字段表单
- 现有弹窗保留给复杂操作更合理

### 决定 2：挂单类型自动判断，而不是要求用户手选

原因：

- 用户已明确要根据价格线和当前价的相对位置自动判断
- 这可以减少一个显式选择步骤
- 也更符合图表内拖线挂单的心智模型

### 决定 3：快捷入口继续复用现有执行链

原因：

- 避免引入第二套交易提交流程
- 保证检查、提交、回执、刷新口径一致
- 可以用更少的改动得到完整闭环
