# UI 系统化重构说明

## 1. 优化后的页面结构说明

| 页面 | 新结构 | 说明 |
|---|---|---|
| 首页 `activity_main` | 页面标题区 -> 监控工作台 -> 行情概览 -> 最近异常记录 -> 异常判断配置 -> 底部导航 | 先状态、再数据、再记录、再配置，主次路径更清楚 |
| 列表页 `activity_log` | 顶部返回+标题区 -> 批量操作区 -> 日志列表 | 去掉视觉噪音，操作按钮集中，列表扫描更快 |
| 详情页 `activity_account_stats` | 页面标题区 -> 账户总览 -> 净值/结余曲线 -> 收益统计表 -> 当前持仓 -> 交易记录 -> 交易统计 -> 底部导航 | 按“概览 -> 趋势 -> 明细 -> 统计”排序，信息流更自然 |

## 3. 设计规范总结

### Spacing

| 项目 | 规范 |
|---|---|
| 页面左右留白 | 16dp |
| 卡片与卡片间距 | 16dp |
| 卡片内边距 | 16dp |
| 组件间距 | 8dp / 12dp |
| 微间距 | 4dp |

规则：
- 模块间距大于组件间距
- 同类控件统一高度
- 列表项统一底部间距

### Typography

| 层级 | 规范 |
|---|---|
| 页面标题 | 24sp / `sans-serif-medium` |
| 模块标题 | 18sp / `sans-serif-medium` |
| 次级标题 | 14sp / `sans-serif-medium` |
| 正文 | 14sp / `sans-serif` |
| 辅助信息 | 12sp / `sans-serif` |

### Components

| 组件 | 规范 |
|---|---|
| 卡片 | 白底、14dp 圆角、1dp 描边、轻阴影 |
| 次级按钮 | 白底/浅底、10dp 圆角、1dp 描边、40dp 高 |
| 选中按钮 / 状态 Chip | 蓝底、白字、10dp 圆角 |
| 输入框 / 筛选框 | 浅底、10dp 圆角、1dp 描边、40dp 高 |
| 指标卡 | 统一内边距、统一字级、统一背景 |
| 底部导航 | 56dp 高，选中态与未选中态区分明确 |

### 颜色

| 类型 | 用途 |
|---|---|
| 主色 `accent_blue` | 关键选中态、主操作、导航当前项 |
| 辅助蓝 `accent_cyan` | 图表和次级强调 |
| 正向色 `accent_green` | 盈利、上涨 |
| 负向色 `accent_red` | 亏损、错误、风险 |
| 中性色 | 页面背景、卡片、边框、辅助文字 |

## 4. 优化前后对比说明

| 原问题 | 优化前 | 优化后 |
|---|---|---|
| 信息层级混乱 | 页面一上来就进卡片，缺少明确标题区 | 首页/列表/详情都增加标题区，先交代“这是哪里、能做什么” |
| 间距不统一 | 10、14、16 混用，模块和组件距离接近 | 统一为 4/8/12/16 体系，模块间距固定 16 |
| 控件尺寸不一致 | 32/34/36/48 高度混杂 | 主要交互控件统一到 40，高频底部导航 56 |
| 风格不统一 | 复古风残留，标题 serif，按钮和卡片语言不一致 | 收敛为现代产品界面语言，统一 sans-serif、统一圆角和描边 |
| 视觉重心分散 | 首页状态、行情、配置都像同等级块 | 改为“状态 -> 数据 -> 记录 -> 配置”的稳定视觉顺序 |
| 列表筛选拥挤 | 交易记录 3 个筛选器一排挤在一起 | 改为两行布局，产品/方向一行，排序单独一行 |
| 选中态不明显 | 多处选中态靠微弱色差区分 | 统一改成蓝底白字，当前状态一眼可见 |
| 列表项样式不齐 | 指标卡、日志卡、持仓卡、交易卡各有各的间距和字号 | 统一卡片边框、圆角、内边距和文本层级 |
