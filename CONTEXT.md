## 近期关键决定和原因
- 部署入口统一为韩国服务器单地址 `http://43.155.214.62:8787`，同时承接 MT5、Binance REST 和 Binance WebSocket；原因是当前路径最短，手机端配置也最简单。
- Binance 行情链路从“手机直连官方地址”改为“运行时跟随设置页网关地址，经服务器 `/binance-rest` 和 `/binance-ws` 转发”；原因是用户要求主监控页和图表页全部走韩国服务器。
- MT5 地址配置从“BuildConfig 固定值”升级为“BuildConfig 默认值 + 设置页运行时覆盖”；原因是服务器入口可能继续变动，不能每次都重新打包 APK。
- 设置页改为“微信式目录首页 + 二级详情页”；原因是用户希望先看设置类别，再进入下一级操作，而不是把所有开关堆在一个页面。
- 主题系统恢复并升级为 5 套完整主题，且桌面图标跟随主题切换；原因是用户后续明确要求真正的多主题系统，而不是单一深色方案。
- 悬浮窗数据改为“统一快照一次性刷新”；原因是用户指出总盈亏、连接状态、价格、成交量更新时间不一致，需要让所有可见数据同批更新。
- 悬浮窗交互改为“整块区域长按拖动，最小化后也能拖动且点击可还原”，同时对拖动过程增加节流；原因是旧实现只有局部可拖且拖动卡顿。
- 悬浮窗内容格式改为：顶部合并盈亏、第二行连接状态、下方按产品显示价格/成交量/成交额和分产品盈亏；原因是用户希望后台一眼看到账户总盈亏和每个产品情况。
- 账户统计页月收益表改为“左侧年份整块 + 右侧 1-6 月 / 7-12 月双行月份 + 横向滚动”；原因是旧版年份和月份对不齐，且百分比文本会被压缩隐藏。
- 账户统计页净值/结余曲线新增最大回撤区间高亮；原因是用户希望在任何周期下都能直接看到回撤最深区段。
- 异常交易提醒先升级为 App 端更强的系统通知；原因是用户当前最关心提醒可靠性，至于“异常判断是否迁到服务器”留到后续服务端优化阶段再做。
- `MA / EMA` 默认关闭；原因是用户要求图表初始界面更简洁。
- 图表历史、历史交易、账户摘要、持仓快照统一走 Room 持久层；原因是用户要求只要拉过的数据就长期保留，并且支持分项清理。
- 2026-03-31 联调时，公网网关接口可正常返回数据；如果某一时刻 `positions=[]`，则 APP 的“暂无持仓”属于真实账户状态，而不是客户端渲染故障。
- 2026-03-31 本轮新增决定：悬浮窗改为更紧凑的“纯文本行式布局”，不再使用产品卡片；原因是用户要求减少拖动卡顿、缩小体积，并让盈亏数字与价格右侧更整齐。
- 2026-03-31 本轮新增决定：账户总览概览指标与“当前持仓”统一使用同一份持仓列表重算；原因是旧实现会出现持仓明细已刷新，但顶部“持仓盈亏/持仓收益率”仍滞后的问题。
- 2026-03-31 本轮新增决定：状态栏和导航栏颜色改为跟随主题实时设置浅深图标；原因是原先不同主题下状态栏文字和图标经常看不清。
- 2026-03-31 本轮新增决定：悬浮窗拖动从“长按后拖动”改为“任意位置直接拖动，点击产品区才跳转”；原因是用户希望拖动更直接，同时保留点击进入对应产品行情页。
- 2026-03-31 本轮新增决定：交易记录排序切换时主动清空展开状态；原因是旧实现会把旧排序下展开的明细带到新排序结果里，视觉上很乱。
- 2026-03-31 本轮新增决定：悬浮窗最小化切换改为“先隐藏图标并立即重排，再显示图标”；原因是旧实现会先在左侧闪一下，再跳到右侧。
- 2026-03-31 本轮新增决定：账户概览下一次刷新时间只在本次快照真正结束后再排队；原因是旧实现会把上一轮计划中的 5 秒/7 秒倒计时继续显示到下一轮里，用户看到的刷新秒数和真实刷新节奏不一致。
- 2026-03-31 本轮新增决定：行情监控页与图表页顶部产品选择标签统一补下拉箭头，并按当前主题染色；原因是用户指出纯文本标签看不出可以点击展开。
- 2026-03-31 本轮新增决定：账户概览标题统一只显示数字账号；原因是用户要求顶部账号信息稳定，不要在数字账号和英文账户名之间切换。
- 2026-03-31 本轮新增决定：悬浮窗最小化切换时先临时隐藏整个浮层，再按新尺寸更新窗口参数后显示；原因是仅切子视图可见性仍可能被肉眼看到中间错位帧。

- 2026-03-31 本轮新增决定：悬浮窗展开态产品行宽度进一步收紧为“整体宽度 110dp + 左右各 5dp 留白 + 产品名列 20dp + 盈亏列 76dp + 内容行 96dp”；原因是用户继续反馈盈亏数字仍偏右、右侧空白仍偏大，需要再压缩一档。
- 2026-03-31 本轮新增决定：悬浮窗头部产品名行改为“头部整行宽度与数值行一致 + 产品名列 28dp + 盈亏列 64dp”；原因是用户要求产品名不换行且必须完整显示，同时盈亏列还要继续左移。
- 2026-03-31 本轮新增决定：账户统计页在消费预加载快照前，若发现预加载快照不带交易记录，则自动回填本地已留存交易；原因是预加载链路为了提速走的是轻量快照，历史交易并没有丢，但旧页面优先用了这份不完整快照，导致首屏先空白。
