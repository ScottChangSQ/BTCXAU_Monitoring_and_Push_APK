## 近期关键决定和原因
- 部署层统一改为“单公网入口 + 路径分流”：`/mt5` 给 MT5 Python 网关，`/binance-rest` 给 Binance REST，`/binance-ws` 给 Binance WebSocket；原因是当前只有一台香港 Windows 主机和一个公网 IP，这条路最短。
- MT5 网关示例环境改为默认监听 `127.0.0.1:8787`，避免把内部网关端口直接暴露到公网。
- `start_gateway.ps1` 改为显式使用本地 `.venv\Scripts\python.exe`，并支持从指定 `.env` 文件加载环境变量，减少部署时“脚本能启动但环境没生效”的隐性问题。
- 现阶段不改 App 业务代码，因此 Binance 仍保留官方地址安全回退；服务器代理先作为主入口，优先保证可部署和可用。
- 额外新增 `deploy/tencent/windows_server_bundle` 作为服务器上传包，而不是继续压缩整仓库；原因是目标服务器只需要网关、部署脚本和代理样例，这样传输更快、操作也更简单。
- MT5 网关新增 `/v1/summary` 轻量摘要接口，后台预加载默认走摘要；原因是当前流量大头来自 `range=all` 全量账户快照。
- 图表页进入前台时，会临时把账户预加载切回全量；原因是图表持仓标注和挂单信息仍依赖完整持仓数据，不能只看摘要。
- 历史行情缓存边界已确认：只要拉取过就一直保留，不设自动上限；原因是用户优先要减少重复流量并保留完整本地历史。
