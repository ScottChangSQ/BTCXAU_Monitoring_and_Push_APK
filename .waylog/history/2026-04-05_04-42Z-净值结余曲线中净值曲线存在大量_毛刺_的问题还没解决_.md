`当前区间仓位`
公式是：`仓位 = 保证金 / 当前净值`

当前页面显示值直接取 `CurvePoint.positionRatio`，见 [AccountStatsBridgeActivity.java](/E:/Github/BTCXAU_Monitoring_and_Push_APK/app/src/main/java/com/binance/monitor/ui/account/AccountStatsBridgeActivity.java#L3697) 和 [AccountStatsBridgeActivity.java](/E:/Github/BTCXAU_Monitoring_and_Push_APK/app/src/main/java/com/binance/monitor/ui/account/AccountStatsBridgeActivity.java#L3701)。
这个 `positionRatio` 的生成规则是：

1. 服务端如果已经传了历史 `positionRatio`，页面直接用。
2. 如果没传，客户端优先按历史成交回放：
   `每个时点仓位 = 当时仍在持仓的保证金合计 / 当时净值`
   见 [AccountCurvePositionRatioHelper.java](/E:/Github/BTCXAU_Monitoring_and_Push_APK/app/src/main/java/com/binance/monitor/ui/account/AccountCurvePositionRatioHelper.java#L83) 到 [AccountCurvePositionRatioHelper.java](/E:/Github/BTCXAU_Monitoring_and_Push_APK/app/src/main/java/com/binance/monitor/ui/account/AccountCurvePositionRatioHelper.java#L89)。
3. 再不行才用当前持仓兜底：
   `仓位 = 当前持仓保证金合计 / 当前净值`
   其中当前持仓保证金合计约等于
   `Σ(abs(marketValue)) / leverage`
   见 [AccountCurvePositionRatioHelper.java](/E:/Github/BTCXAU_Monitoring_and_Push_APK/app/src/main/java/com/binance/monitor/ui/account/AccountCurvePositionRatioHelper.java#L55) 到 [AccountCurvePositionRatioHelper.java](/E:/Github/BTCXAU_Monitoring_and_Push_APK/app/src/main/java/com/binance/monitor/ui/account/AccountCurvePositionRatioHelper.java#L63) 和 [AccountCurvePositionRatioHelper.java](/E:/Github/BTCXAU_Monitoring_and_Push_APK/app/src/main/java/com/binance/monitor/ui/account/AccountCurvePositionRatioHelper.java#L189) 到 [AccountCurvePositionRatioHelper.java](/E:/Github/BTCXAU_Monitoring_and_Push_APK/app/src/main/java/com/binance/monitor/ui/account/AccountCurvePositionRatioHelper.java#L200)。

   - 优点：界面可以一次做干净，交互更统一。
   - 缺点：工作量明显更大，短期内不能最快解决“不要命令行”。

3. **做 Windows 本地桌面壳，里面包 Web 控制台**
   - 做法：服务器本机运行一个桌面程序，内部再嵌 Web 面板。
   - 优点：更像“本地软件”，心理上更接近运维工具。
   - 缺点：比纯 Web 多一层壳，维护更复杂；远程使用反而不如浏览器方便。
